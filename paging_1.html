#define PMD_SHIFT (PAGE_SHIFT + (PTRS_PER_PTD_SHIFT))
We will simply go through as much of the paging.h file as we can, (arch ia64), So what is PAGE_SHIFT? Well, it depends on the configuration, and the configuration, like many things, is best seen through a table.

Configuration PAGE_SHIFT value
CONFIG_IA64_PAGE_SIZE_4KB 12
CONFIG_IA64_PAGE_SIZE_8KB 13
CONFIG_IA64_PAGE_SIZE_16KB 14
CONFIG_IA64_PAGE_SIZE_64KB 16

So clearly the page size or some fragment of that value is being set, but under what context? virtual? physical? It's best to see where these configuration parameters are being set. These are referenced in the Kconfig file, which makes sense, but when is this file read? Well this has to do with the build process and its beyond the scope of this topic, what is important is that these configuration parameters are set very early, and therefore have a type of authority. Continuing to PTRS_PER_PTD_SHIFT, it expands to (PAGE_SHIFT-3, which means the entire expression expands to PAGE_SHIFT + (PAGE_SHIFT-3) I'm tempted to assume that this is equivalent to (2 * PAGE_SHIFT) - 3, though I am not sure why they do this, but it does look strange from an arithmetic perspective. Equally interesting along this observation is this line here hmm_pfn_flags_order(PMD_SHIFT - PAGE_SHIFT); Because if true, this would be equivalent to hmm_pfn_flags_order(PTRS_PER_PTD_SHIFT);

Tracing programs, maybe gdb


Anyway, so lets trace some syscalls with strace, lets trace gdb. First of all, running the command strace gdb will make gdb unuseable, it will essentially just completely take over standard output. The better way to go about it is strace --attach=<pid>, then you will be privy to the sea of output from strace. So first of all we know that we want to trace only the write() system call, so maybe strace can be filtered on its system calls.
